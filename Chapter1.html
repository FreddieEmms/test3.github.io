User Challenge: Can you now write a basic program using the Console.Writeline(“”).

To run your program, you can press F5 or you can click run program. Now if you are unfortunate, it may pop up that there is a system error and the program cannot run. This means that the program cannot understand your code as you may have made a mistake. Programs tend to be very strict about what you write and even if there is the smallest error it will not run. Fortunately, your code will be highlighted by an IDE and show that there is an error on that line. One common error is not including the semi-colon which instructs the program that a new command is being entered. Now, if an error did occur, change your code to the example above and then it should be fixed.

Now that you've fixed your code, for some users, it may quickly flash a box and then nothing happens. This is because we have no input into our program and if there is no input from the user then it will quickly print our code and shut down because it does not require us. To stop this we can use Console.Readline();. This means that a computer must wait for a response from the user (which in this case an entry to make us continue). This can be used multiple times to add pauses and soon we will use it to assign inputs to variables!

Another way to write your code is using Console.Write("…"); , now instead of writing the code onto a new line each, this will make it so that it will be written on the same line. Try these new techniques out!
